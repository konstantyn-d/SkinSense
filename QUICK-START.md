## 🎯 Что можно тестировать СЕЙЧАС

### 1. Запуск серверов

**Backend** (уже запущен):
```bash
cd backend
npm run dev
# Работает на: http://localhost:4000
```

**Frontend** (уже запущен):
```bash
cd frontend
npm run dev
# Работает на: http://localhost:5173
```

### 2. Тестирование

Откройте браузер: **http://localhost:5173**

#### ✅ Регистрация
1. Нажмите на вкладку **"Register"**
2. Заполните форму:
   - Full Name: `Ваше имя`
   - Email: `test@example.com`
   - Password: `password123` (минимум 6 символов)
   - Confirm Password: `password123`
3. Нажмите **"Create Account"**

**Возможные результаты:**
- ✅ Успешная регистрация → автоматический вход → переход на `/profile`
- ✅ Или сообщение "Please check your email" (если включено подтверждение email в Supabase)

#### ✅ Вход
1. Вкладка **"Login"**
2. Введите email и password
3. Нажмите **"Login"**
4. Должны попасть на страницу профиля

#### ✅ Навигация
Используйте нижнюю панель навигации:
- **👤 Profile** - профиль пользователя
- **📸 Scan** - страница сканирования (заглушка)
- **📈 Progress** - отслеживание прогресса (заглушка)

#### ✅ Выход
На странице Profile нажмите кнопку **"🚪 Logout"**

### 3. Что вы должны увидеть

#### На странице Profile:
- ✅ Ваш аватар с инициалами
- ✅ Имя и email
- ✅ Статистика (пока всё 0)

- ✅ Кнопка "Take Your First Scan"

## 🐛 Возможные проблемы

### "Missing Supabase environment variables"
**Решение:** Проверьте `frontend/.env`:
```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_API_BASE_URL=http://localhost:4000/api
```

### "Unable to connect to backend"
**Решение:** Убедитесь что backend запущен:
```bash
cd backend
npm run dev
```

### Регистрация не работает
**Решение:** 
1. Проверьте Supabase Dashboard > Authentication > Providers
2. Убедитесь что Email provider включен
3. Попробуйте другой email

### 401 Unauthorized при API запросах
**Решение:**
1. Выйдите и войдите снова
2. Проверьте консоль браузера на ошибки токена
3. Очистите localStorage: `localStorage.clear()`

## 📊 API Endpoints (готовы к использованию)

### Без аутентификации:
- `GET /api/health` - проверка работоспособности

### С аутентификацией (Bearer token):
- `GET /api/scans` - список сканов
- `GET /api/scans/:id` - детали скана
- `POST /api/scans` - создать скан (с изображением)
- `GET /api/progress/summary` - сводка прогресса
- `GET /api/progress` - список активных проблем
- `PATCH /api/progress/:id` - обновить прогресс
